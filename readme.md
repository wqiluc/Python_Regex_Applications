<h3>📄 contrato_usando_regex.py</h3>
<p align="left">
Neste exemplo, aplico Expressões Regulares para identificar cláusulas específicas,
valores monetários, datas contratuais e padrões formais dentro de documentos.
O foco é demonstrar como o Regex pode automatizar leitura e validação de contratos.

```python
(f"""
 Desenvolva um programa que pergunte para o usuário: 
o nome completo, 
cpf, 
endereço, e 
duração do contrato. 
Personalize todo o texto do contrato de acordo com as informações dadas. """)

#solução:👇
from cores import *
from re import fullmatch as padrao

lista_dados_cliente = [ ]

nome_cliente = str(input(f"{Negrito}Digite seu nome completo: {Reset}")).strip().capitalize()
while not (nome_cliente.replace(" ", "", 10).isalpha()):
    print(f"{Vermelho}Termo Inválido!! ❌ Digite seu NOME Completo (str){Reset}")
    nome_cliente = str(input(f"{Negrito}Digite seu nome completo: {Reset}")).strip().capitalize()
nome_cliente = str(nome_cliente)
lista_dados_cliente.append(nome_cliente)

cpf_cliente = str(input(f"{Negrito}Digite o CPF do cliente. Ex: (XXX.XXX.XXX-XX): {Reset}").strip())
while not (padrao(r"\d{3}\.\d{3}\.\d{3}-\d{2}", cpf_cliente)):
    print(f"{Vermelho}CPF inválido! Use o formato XXX.XXX.XXX-XX{Reset}")
    cpf_cliente = str(input(f"{Negrito}Digite o CPF (XXX.XXX.XXX-XX): {Reset}").strip())
lista_dados_cliente.append(cpf_cliente)

duracao_contrato = str(input(f"{Negrito}Digite a data que o contrato terminará (DD/MM/AAAA): {Reset}").strip())
while not (padrao(r"\d{2}/\d{2}/\d{4}", duracao_contrato)):
    print(f"{Vermelho}Data inválida! Digite no formato DD/MM/AAAA{Reset}")
    duracao_contrato = str(input(f"{Negrito}Digite a data que o contrato terminará: {Reset}").strip())
lista_dados_cliente.append(duracao_contrato)

print(lista_dados_cliente)

print("\n" + "=" * 80)
print(f"{'CARTILHA CONTRATUAL':^80}")
print("=" * 80)

print(f"\n{Negrito}Contratante: {nome_cliente}{Reset}")
print(f"{Negrito}CPF: {cpf_cliente}{Reset}")
print(f"{Negrito}Data de término do contrato: {duracao_contrato}{Reset}")

print("\n" + "-" * 80)
print(f"{Negrito}CLÁUSULA 1ª – DO OBJETO{Reset}")
print(f"O presente contrato tem como finalidade formalizar o acordo firmado entre as partes.")

print(f"\n{Negrito}CLÁUSULA 2ª – DA VIGÊNCIA{Reset}")
print(f"{Negrito}O presente contrato terá validade até {duracao_contrato}.{Reset}")

print(f"\n{Negrito}CLÁUSULA 3ª – DAS RESPONSABILIDADES{Reset}")
print(f"{Negrito}As partes comprometem-se a cumprir integralmente as condições estabelecidas neste instrumento.{Reset}")

print("\n" + "-" * 80)
print(f"\n{Negrito}Assinam o presente instrumento:{Reset}\n")

print(" " * 5 + "_" * 35 + " " * 15 + "_" * 35)
print(" " * 8 + f"Contratante - {nome_cliente}" + " " * 13 + "Contratado(a)")

print("\n" + "=" * 80)
```

<h3>📧 email_usando_regex.py</h3>
<p align="left">
Validação estrutural de e-mails utilizando padrões robustos.
Aqui são tratados caracteres permitidos, domínio, subdomínios e verificação
com âncoras para garantir que o formato seja válido.

```python
(f"""
Crie um programa que dado o e-mail de um usuário, 
valide se aquele e-mail é válido com as seguintes regras:
    - Tem que ter .com ou .gov;
    - Tem que ter @;
    - Tem que ter mais de 5 caracteres; e
    - Não pode ter espaço. """)

#solução:👇
from cores import *
from re import fullmatch as padrao

email_usuario = str(input(f"{Negrito}Digite o email do usuário: {Reset}")).strip()

    print(f"{Vermelho}Termo Inválido!! ❌ Digite um padrão VÁLIDO para o email. Siga as regras acima 👆{Reset}")
    email_usuario = str(input(f"{Negrito}Digite o email do usuário: {Reset}")).strip()
```

```python
from cores import *
```

```python
 Analisador de URL

{Amarelo}https://www.hashtagtreinamentos.com/blog{Reset}{Magenta}

    - Domínio:{Reset} {Amarelo}hashtagtreinamentos.com{Reset}{Magenta};
```

<h3>🪪 cpf_avançado_usando_regex.py </h3>
<p align="left">
Validação estrutural e mascaramento de CPF com Regex. O script identifica o formato brasileiro (123.456.789-00), utiliza grupos de captura para separar blocos numéricos e aplica substituição parcial para proteção de dados sensíveis. Demonstra uso prático de captura, referência a grupos e re.sub().

```python

print(f"""
╔══════════════════════════════════════════════════════╗
║              🪪 MASCARADOR DE CPF 🇧🇷 🎭               ║
╠══════════════════════════════════════════════════════╣

🎯 Objetivo:
Identificar e mascarar CPFs em um texto garantindo:

• Reconhecimento do formato 123.456.789-00;
• Validação estrutural do padrão brasileiro;
• Substituição segura dos 9 primeiros dígitos;
• Preservação dos 2 dígitos verificadores;
• Tratamento automatizado via Regex;
• Proteção de dados sensíveis;

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📌 Exemplo tratado pelo algoritmo:

Entrada:
CPF do cliente: 123.456.789-00

Saída:
CPF do cliente: ***.***.***-00

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🧠 Conceitos aplicados:
Regex (re);
Grupos de captura ();
Referência a grupos (\\4);
Escape de caracteres especiais (\\.);
Quantificadores {3} e {2};
Substituição com re.sub(); e
Mascaramento parcial de padrões sensíveis.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🚀 Sistema preparado para anonimização estrutural de CPFs """)

#resolução👇🎭:
from cores import *
import re as padrao
cpf = str(input(f"\n {CinzaClaro}Digite o seu CPF: (ex: XXX.XXX.XXX-XX){Reset}")).strip()
padrao_cpf = (r"(\d{3})\.(\d{3})\.(\d{3})-(\d{2})")
while not (padrao.fullmatch(padrao_cpf,cpf)):
    print(f"{Vermelho}Termo Inválido!! ❌ Digite um CPF (ex: XXX.XXX.XXX-XX){Reset}")
    cpf = str(input(f"\n {CinzaClaro}Digite o seu CPF: (ex: XXX.XXX.XXX-XX){Reset}")).strip()
    padrao_cpf = (r"(^\d{3})\.(\d{3}\.{3})-(\d{2}$)")

resultado = padrao.sub(padrao_cpf,(r"***.***.***-\4"),cpf)

print(f"\n {CinzaClaro}O cpf Mascarado é:{Reset} {Amarelo}{resultado[0:12]}{Reset}{CinzaClaro}{resultado[12:]}{Reset}")
```

<h2 align="center">🧠🔥 Conclusão: </h2>

As **Expressões Regulares (Regex)** são uma das ferramentas mais poderosas da programação quando o assunto é **validação, extração, substituição e manipulação de texto**.

Nos exemplos apresentados (CPF, CEP, telefone, placa Mercosul, e-mail, URL e contratos), você aplicou Regex para:

- Garantir **validação estrutural exata**;
- Utilizar **âncoras (^ e $)** para impedir entradas parciais;
- Trabalhar com **grupos de captura ()**;
- Aplicar **quantificadores ({}, +, ?, *)**;
- Usar **classes de caracteres ([A-Z], \d, \w)**;
- Realizar **substituições seguras com re.sub()**; e
- Automatizar processos que manualmente exigiriam dezenas de condicionais.

---

## 🧩 O impacto prático:

Sem Regex, você precisaria criar múltiplos `if`, `len()`, `split()`, verificações manuais de caractere por caractere e validações fragmentadas.

Com Regex, você descreve um padrão matematicamente e valida tudo em **uma única expressão estruturada**.

Isso gera:

- Redução de complexidade;
- Maior legibilidade (quando bem escrito);
- Código mais enxuto;
- Melhor performance em validações estruturais; e
- Padronização profissional.
